Enumerable Methods: Map and Collect

December 14th, 2015
Jeff Schneider

What's an Enumerable?

One of the things that makes Ruby powerful, is being able to use Enumerable methods. “Enumerable” is a “module”. Simply stated, that means it a bunch of methods packaged together that can be “mixed in” with other classes (like Array and Hash).

This means that you don’t have to write out every method many different times. You can write them just once, package them up as Enumerable, and then tell Array and Hash to use them. The only catch is that the class that wants to use Enumerable has to have its own #each method that Enumerable can use.

Enumerable provides lots of useful ways of doing things to elements of a collection object (like an Array or Hash). For example, you are probably familiar with the #each iterator method that comes pre-packaged with the Array and Hash classes. It basically goes through each element of the object and passes it to a block you’ve specified: Returning: That should look familar. Now let’s take a look at something a bit more complicated to see why we would want to use an Enumerable method instead of #each.

Let’s say we want modify all of the elements in an Array (say take them to the third power)? Just using the #each method, we would have to use a complicated loop: Returning:

Enumerable methods to the rescue!

You can use either the map or collect (they do the same thing) Enumerable methods. Array.map{|x| x**3}

Using the map method, we’ve refactored a complicated loop into a single line. It’s that simple. The map and collect methods return a new array with the result of running a block once for every element.
