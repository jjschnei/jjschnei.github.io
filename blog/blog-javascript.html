Data Structure Smackdown: Objects vs. Hashes!

January 3rd, 2016
Jeff Schneider

Ruby Hashes vs. JavaScript Objects

If you have had some exposure to Ruby and JavaScript, you’ve likely noticed that Ruby’s hashes and JavaScript’s objects seem related. In this post, I’ll first explore their similarities, followed by the differences.

Similarities

Hashes and objects accomplish the same general thing. They are both structures that hold values which are by keys. All keys must be unique, however, values can be repeated. Both data structures look similar. Here is how you would create a hash and object using literal notation: 


        //JavaScript object:
        var jeff = { from: “Porltand”, favorite_food: “Burritos”}

        #Ruby hash:
        jeff = { from: “Porltand”, favoriteFood: “Burritos” };

In both data structures, you use the name of the data structure with its key or property to get the matching value. Both hashes and objects accept several syntax variations to access the values. Let’s take a look at an example: 


        //JavaScript object:
        jeff[“from”];
        => Portland

        #Ruby hash:
        jeff[:from]
        => Portland

As you will see they look very similar. Once you’ve created a hash or object, both data structures can be similarly manipulated. For example, you can add more key / value pairs. You can remove key / value pairs. You can even make both data structures multidimensional.

Differences

There is a difference in terminology between objects and hashes. While the data in hashes are called keys and values, in JavaScript the pairs are often referred to as ‘properties’. This difference is in name only as they do the same thing. A more fundamental difference is that you can put a function inside of an object, but not in a hash.

The hash and the object have battled evenly to this point. However, the ability to put a function in an object gives it more functionality than a hash (see what I did there?).
