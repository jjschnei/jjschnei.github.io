A Basic Overview of Recursion

January 10th, 2016
Jeff Schneider

What is Recursion?

Recursion is a method where the solution to a problem is found by solving smaller instances of the same problem. It’s a helpful technique to use when problems are too big or complex to solve directly. By breaking down a difficult problem into smaller versions of itself, it becomes possible to solve one of these smaller versions and then build up to a solution of the entire problem.

There are several types of recursion:

  • Single recursion and multiple recursion
  • Indirect recursion
  • Anonymous recursion
  • Structural versus generative recursion
The specifics of each is beyond the general overview I’m providing here. You can learn more about the intricacies of each type on Wikipedia.

Recursion is one of the most deceptively simple programming concepts. It sounds straightforward, but once you start working with it and try to really understand how it works, you may find yourself dizzy and confused. If you find yourself feeling this way, take comfort knowing that the more you try it the easier and more comfortable you will become with the concept.

When Would You Use Recursion?

In general, problems that are defined in terms of themselves are good candidates for recursive techniques. When using recursion, every recursive call must simplify the computation in some way. There must be special cases to handle the simplest computations – these are called the base cases.

The base case, or halting case, of a function is the problem that we know the answer to. It can be solved without any more recursive calls. The base case is what stops the recursion from continuing on forever. Every recursive function must have at least one base case.

Example of Recursion

Let’s take a look at the standard example used by many computer science textbooks, the factorial function. The factorial of a number n is calculated by multiplying 1 * 2 * 3 *... n. Here is how you would solve the problem recursively in JavaScript: 


          int factorial (int n) {
              if (n == 1) {
                  return 1;
              } else {
                  return n*factorial(n-1);
              }
          }
It’s important to remember that recursion is not the only way to solve problems. You can often use iteration instead. There are advantages and disadvantages to each approach. Here is how you would solve the above factorial problem using iteration: 


        int factorial (int n) {
            int product = 1;
            for(int i=2; i < n; i++) {
                product *= i;
            }
            return product;
        }
